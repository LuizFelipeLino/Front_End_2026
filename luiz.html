Engenharia de Software: Diagramas UML

O Diagrama de Casos de Uso descreve as funcionalidades do sistema de acordo com usuario. Ele é essencial para entender quem usa o sistema e o que faz.

Elementos Principais

  • Ator: Representa os papéis que interagem com o sistema (ex: Cliente, Admin).
  • Caso de Uso: Representa uma funcionalidade (ex: Fazer Login).
  • Fronteira: Define o limite do sistema.
